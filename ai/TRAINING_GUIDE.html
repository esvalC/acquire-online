1 How It Works

The Master bot uses a value network — a small neural network that predicts "how likely am I to win from this position?" For every legal action on its turn, the bot clones the game, applies the action, encodes the resulting state, scores it with the network, and picks the highest-scoring move.

selfplay.js → games.jsonl → train.py → master_weights.json → masterBot.js
(EC2 runs 5h) (trains on Mac) (inference on server)

The network architecture is a 4-layer MLP with 80,000 parameters:

Input (149) → Dense 256 → Dense 128 → Dense 64 → Output (1) board(108) + chains(35) + cash(6) ReLU ReLU ReLU sigmoid → win %
Why a value network (not a policy network)?
We don't need to capture which action the bot took — just whether it won. A value network learns what game positions are good and then uses one-step lookahead to find the best action. Simpler training data, equally strong results.

2 One-Time Setup

Install Python dependencies (Mac)

pip install torch numpy

Verify AWS access

aws s3 ls s3://acquire-training-data/
No output = no training data yet. Run a selfplay session first (Section 3).

3 Generate Training Data (EC2)

This launches a spot instance that plays bots against each other for 5 hours (~$0.10/hr × 5h = $0.50 per run), then uploads the results to S3 and self-terminates.

1

Launch the training instance

aws ec2 run-instances \
  --image-id ami-0c421724a94bba6d6 \
  --instance-type c5n.xlarge \
  --instance-market-options '{"MarketType":"spot"}' \
  --iam-instance-profile Name=acquire-online-ec2-role \
  --security-group-ids sg-0689315c5c119b6ff \
  --user-data "$(base64 -i ai/ec2_train.sh)" \
  --instance-initiated-shutdown-behavior terminate \
  --tag-specifications \
    'ResourceType=instance,Tags=[{Key=Name,Value=acquire-training}]' \
  --query 'Instances[0].[InstanceId,PublicIpAddress]' \
  --output text
2

Get the instance IP (if it wasn't shown)

aws ec2 describe-instances \
  --filters 'Name=tag:Name,Values=acquire-training' \
            'Name=instance-state-name,Values=running' \
  --query 'Reservations[0].Instances[0].PublicIpAddress' \
  --output text
3

Watch the live dashboard

Open http://<IP>:8080 in your browser. You'll see games/sec, ELO ratings per bot, and time remaining. Wait ~5 minutes for startup.

The instance auto-terminates after 5 hours. You don't need to clean anything up. Check S3 for the output: aws s3 ls s3://acquire-training-data/

4 Train the Neural Network (Mac)

Option A — One command does everything

The retrain.sh script handles download → train → commit → deploy automatically:

bash ai/retrain.sh

Optional flags:

FlagEffect
--skip-downloadUse existing ai/data/games.jsonl instead of fetching from S3
--skip-deployTrain and commit but don't push to the server
--epochs 30Override training epochs (default: 20)

Option B — Manual steps

1

Download training data

aws s3 cp s3://acquire-training-data/latest.jsonl ai/data/games.jsonl
2

Train the model

python3 ai/train.py \
  --data ai/data/games.jsonl \
  --out  ai/models/master_weights.json \
  --epochs 20

Takes ~5 minutes on a Mac. Watch for val_loss decreasing each epoch — lower is better.

3

Commit the weights

git add ai/models/master_weights.json
git commit -m "Train Master bot: 50000 records, val_loss=0.2314"
git push origin main
4

Deploy to server

aws ssm send-command \
  --instance-ids i-0bbc6c13fd3dfe6ab \
  --document-name "AWS-RunShellScript" \
  --parameters 'commands=[
    "sudo -u ubuntu GIT_SSH_COMMAND=\"ssh -i /home/ubuntu/.ssh/deploy_key\" git -C /home/ubuntu/acquire-online pull origin main",
    "sudo -u ubuntu pm2 restart acquire-online"
  ]'

5 Iterating — Making the Bot Stronger

Each training run improves the bot. Here's the cycle:

Launch EC2 → 5h selfplay → retrain.sh → deploy → repeat $0.50 ~50k records ~5 min on Mac live in 30s

Signs of improvement

MetricWhat to look for
val_lossLower is better. First run: ~0.30. Good: <0.25. Great: <0.22.
val_acc% of outcomes predicted correctly. Target: >65%.
Play testStart a game vs Master at /beta — it should feel aggressive and hard to beat.

How to accumulate more data over time

The S3 bucket stores every run. To train on all historical data combined:

# Download all runs and concatenate
aws s3 cp s3://acquire-training-data/ ai/data/all/ --recursive
cat ai/data/all/*.jsonl > ai/data/games_combined.jsonl

# Train on the combined dataset
python3 ai/train.py \
  --data ai/data/games_combined.jsonl \
  --epochs 30 \
  --out ai/models/master_weights.json
More data = stronger bot. After 3–4 runs (~200k records total), val_loss typically drops below 0.22 and the Master bot noticeably outplays the hard heuristic bots.

Adjusting training hyperparameters

FlagDefaultWhen to change
--epochs20Increase to 40–50 when you have >200k records
--hidden256Increase to 512 for >500k records (larger model)
--batch512Increase for speed on large datasets
